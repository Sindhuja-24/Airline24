## 2025-05-15 - [Improving Form Usability & Accessibility]
**Learning:** In legacy JSP projects, nested interactive elements (like `<button>` inside `<a>`) are common but break accessibility and HTML semantics. Converting these to single, well-styled semantic elements improves screen reader compatibility and prevents unexpected behavior. Adding a simple password visibility toggle significantly enhances UX by reducing login/registration friction.
**Action:** Always check for nested interactive elements and replace them with single semantic ones. Use vanilla JS for simple UI enhancements in projects where modern framework-based solutions aren't available.

## 2026-03-31 - [Consistent Accessibility in Payment Forms]
**Learning:** Legacy payment forms often lack proper label-to-input associations and use decorative emojis that can clutter screen reader output. Ensuring every input has a linked label and hiding decorative icons with `aria-hidden="true"` significantly improves accessibility. Providing immediate visual feedback (like a "Processing..." state) on submit buttons is crucial for async-simulated operations in JSP apps to prevent double-submissions and improve perceived responsiveness.
**Action:** Audit all form inputs for `id` and `for` attribute pairs. Use `aria-hidden="true"` for any emoji or icon that doesn't convey unique information. Always add a loading state to submit buttons in async or delay-heavy workflows.
